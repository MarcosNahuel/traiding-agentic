### 2. Bug crítico #1: Proxy `binance.italicia.com` sirve tickers STALE
- **Evidencia**: 8/13 SL recientes con trigger_price ~5% MENOR que executed_price
- **Causa**: proxy devuelve precios delayed durante rallies alcistas
- **Patrón**: siempre igual dirección (trigger < real) = cache/delay, no random
- **Ejemplo más claro**: BTC trigger=$67,966 aparece DOS VECES en trades diferentes
- **Fix aplicado** (commit `f6ba148`):
  - `binance_client.get_price_direct()` bypassa proxy
  - `binance_client.get_price_safe()` direct-first, proxy-fallback, drift logging
  - `_check_stop_losses` y `_emergency_sl_check` usan `get_price_safe`
- **Impacto estimado**: elimina ~60% de SL falsos → +$3-5/semana

### 3. Bug crítico #2: `reconciliation_runs` bloat 210 MB
- **Root cause**: cada row persistía el FULL balance snapshot de Binance (~320 assets)
- **Cada row pesa ~21 KB** × 1440 runs/día × 7d retention = 210 MB
- **Fix aplicado** (commit `f6ba148`):
  - Guardar solo USDT + active trading symbols (3 entries vs 320)
  - `divergence_details` y `actions_taken` = NULL cuando vacíos (most runs)
  - Nuevas rows serán ~1 KB (de ~21 KB) → 20× reducción
- **Cleanup inmediato**: DELETE de rows >3d → 5114 filas borradas, ~104 MB liberados
- **Estado final**: reconciliation_runs 3863 rows (de 8989)
